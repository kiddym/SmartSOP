# 顶栏 IA + 标准文件库→文件夹配置 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把外壳升级到三段式（顶栏 48px + 侧栏 240/64 + 主区），按 Q321 把入口按"内容容器 vs 管理类"切分；把"标准文件库"改名"文件夹配置"并归入顶栏 ⚙▾ 配置组。

**Architecture:** 拆 `AppTopBar.vue` + `AppSidebar.vue` 两个独立组件，`AppLayout.vue` 退回纯编排（`useSidebar` 单例继续承担折叠态）。`AppTopBar` 的搜索框 / 未读徽标按"诚实占位"策略——搜索 `disabled + title="即将上线"`，未读 `v-if="unreadCount>0"` 当前永假。⚙▾ 用 `el-dropdown @command` 派发，命令清单抽成 `defineExpose` 暴露的常量以利契约测试（绕开 [el-dropdown jsdom test] 已知坑）。

**Tech Stack:** Vue 3 `<script setup>` + TypeScript、Element Plus 2.7、`@vueuse/core` 现有 `useSidebar` composable、Vitest + `@vue/test-utils`。

**Gate（每次"跑测试"与收尾，均在 `frontend/`）：** `npx vitest run tests/unit/<file>.spec.ts` 跑单测；Task 7 收尾再 `npm run lint && npm run typecheck && npm run build`（`--max-warnings 0`）+ 浏览器实测。

<script setup lang="ts">
import { computed } from 'vue'
import { useRoute } from 'vue-router'
import { ElIcon, ElMenu, ElMenuItem } from 'element-plus'
import { Document, EditPen } from '@element-plus/icons-vue'

defineProps<{
  collapsed: boolean
}>()

const route = useRoute()

// /procedures/:id, /procedures/:id/edit, /procedures/:id/view 都归"程序库"
// /procedures/drafts 独立
// ⚙ 下的页面（/settings, /settings/fields, /audit-logs, /folders）侧栏不高亮
const activeMenu = computed<string>(() => {
  if (route.path.startsWith('/procedures/drafts')) return '/procedures/drafts'
  if (route.path.startsWith('/procedures')) return '/procedures/library'
  return ''
})

defineExpose({ activeMenu })
</script>

<template>
